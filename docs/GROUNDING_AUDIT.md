# Phase-2 grounding audit protocol (do not skip)

This is the real go/no-go. If retrieval is dishonest here, no downstream cleverness fixes it —
fix retrieval or walk away. Run with `pqe audit <exploration>`.

## The trap
The prose answer is generated by Claude and is only loosely coupled to whether retrieval
actually found support. **Fluency ≠ correctness.** You only catch a miss by opening the cited
chunks — which is why `pqe run`/`pqe audit` always print the evidence, and the heuristics are
labeled as convenience only.

## Three failure modes — all invisible in the prose
1. **Retrieval missed → pretraining fallback.** No relevant chunk found; the model answers from
   its own memory and staples on the nearest chunk. You think you're reasoning over your library;
   you're not.
2. **Citation doesn't support the claim.** The chunk is real and from your library but says
   something weaker/adjacent (e.g. "Htr2c is *expressed* in DMH" cited for a claim that it
   *drives* thermogenesis). The leap is the model's.
3. **Strength inflation.** Source says "correlation, male mice, one condition"; answer says
   "X causes Y." Grounding exists but got overclaimed — the exact failure you must not inherit
   into a grant aim.

## Test design (`eval_questions.yaml`)
- **4–6 questions you already know the answer to**, and know which paper holds it. You cannot
  audit grounding without ground truth. Set `expect: answerable` + `expect_source`.
- **Exactly one negative control** — a question the library demonstrably *cannot* answer
  (`expect: insufficient_evidence`). A healthy system refuses; a broken one confabulates. This
  matters MORE at scale: with hundreds of papers there's almost always a superficially-close
  chunk, so the temptation to confabulate is higher. This single test is the cleanest read on
  honesty.

## How to read `pqe audit` output
For each question it prints a heuristic verdict, then the answer, then the retrieved chunks.
**Ignore the verdict; read the chunks:**
- For each answerable question: did it retrieve the paper you know is right? Do the chunks
  actually support the claim, *at the right strength*, without an unlicensed leap?
- For the negative control: did it refuse, or did it manufacture an answer from a near-miss chunk?

The heuristics (`PASS`/`FLAG`) only check string signals (did the expected source id appear; did
the answer contain a refusal phrase). They are a triage aid, not the judgment.
